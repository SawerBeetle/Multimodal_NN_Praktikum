# Multimodal_NN_Praktikum

Итоговое задание по модулю 4. **Цель работы** – создать нейросеть, прогнозирующую калорийность блюда по его фото и составу ингредиентов. **Целевая метрика**, по достижении которой обучение считается законченным – MAE (mean absolute error, средняя абсолютная ошибка) тестовой выборки 50 и менее. 

**Данные** включают две таблицы (`data/ingredients.csv` и `data/dish.csv`) и папку с фотографиями. Основные данные содержатся во второй таблице. Это `dish_id` (ID блюда, по которому также извлекаются фото), `total_calories` (общее число калорий, *целевая переменная*), `total_mass` (масса блюда), `ingredients` (список всех ID ингредиентов в формате `ingr_0000000122;ingr_0000000026;..`, где ненулевая часть соответствует ID ингредиента из `data/ingredients.csv`) и `split` (метка, указывающая, куда отнести блюдо, `train`/`test`). Первая таблица содержит сведения об ингредиентах: ID (`id`) и название (`ingr`). Всего к обработке предложено 3262 объекта (фото + список ингредиентов). 

Имеющиеся данные я посчитал необходимым несколько доработать: 

- вместо общей калорийности я рассчитал калорийность на грамм и обучал модель предсказывать именно это значение; 
- собственно калорийность рассчитывалась отдельно через калорийность на грамм и массу, а уже по этим данным рассчитывалась MAE; 
- ID ингредиентов я заменил названиями; 
- обучающий набор (`train` в `dish.csv['split']`) разделил на собственно обучающий и валидационный. 

## Разведочный анализ

Согласно задаче, нужно было загрузить данные, частично их визуализировать и изучить. На основе изучения нужно было определиться с общим ходом работы, предобработкой и метриками. 

Общая структура мультимодальной модели повторяет то, чему учили в Модуле 4: обработка текста проводится `bert_base_uncased`, обработка фото `tf_efficientnet_b0`. Далее выходы моделей приводятся к одной размерности, перемножаются (*хотя можно рассмотреть и конкатенацию*) и прогоняются через небольшую нейросеть-регрессор с единственным нейроном на выходе. Активация ReLU, т.к. отрицательные значения калорийности не рассматриваем. 

В качестве предобработки текста выполнена токенизация названий ингредиентов (замена ID на них проведена на предыдущем этапе). Фотографии из обучающего набора аугментированы средствами из `albumentations`: изменения цветности и контрастности (`RandomBrightnessContrast`, `ToGray` либо `HueSaturationValue`) и дропаут (`CoarseDropout`). Вероятность аугментации не стопроцентная. 

Единственно разумный выбор метрики, исходя из постановки задачи, это `nn.L1Loss`. 

Собственно для разведочного анализа я использовал средства `ydata-profiling`. Данные по результатам разведочного анализа показали хорошее качество. Удалить пришлось единственный объект, для которого была указана масса в 3+ кг, а на фото его вес выглядел раз в 10 меньше. 

## Реализация пайплайна

В этом разделе нужно было подготовить датасеты и загрузчики для них, создать модель, написать функции её обучения и валидации и обеспечить воспроизводимость результатов. 

По итогу: 

- создана мультимодальная модель; 
    - за обработку фото отвечает `tf_efficientnet_b0`; 
    - за обработку текста отвечает `bert-base-uncased`; 
    - последние слои этих моделей дообучаются; 
    - внутри модели создана структура для объединения результатов двух предобученных моделей и небольшая модель для выполнения регрессии; 
- созданы загрузчики и датасеты для обучения, валидации и тестирования; 
- предобработка данных включена внутрь датасетов; 
- созданы функции для обучения и валидации; 
- создан конфиг; 
- обеспечена воспроизводимость результатов (частично; я не стал задействовать детерминированные алгоритмы, чтобы не утяжелять обработку); 
- метрики, описывающие прогресс обучения (лосс и MAE) выводятся на дисплей. 

Нужно отметить, что, возможно, реализованы не все средства обеспечения воспроизводимости. От использования же детерминированных алгоритмов я отказался сознательно, чтобы не затруднять расчёты. 

Ещё один момент, который нужно отметить: обработка проходит достаточно тяжело. Возможно, `tf_efficientnet_b0` стоит заменить на менее точную, но более лёгкую в расчётах модель. 

Также для облегчения тестирования кода добавлено уменьшение количества обрабатываемых объектов и увеличение граничного значения целевой метрики в режиме `prelimininar`. Объектов во всех наборах оставлено по 16, а граничное значение MAE увеличено до 180. Это позволяет в разумные сроки проверить работу кода на локальной маломощной машине. 

Конфигурация прописана в файле `config.yaml`, код сохранён в скриптах `dataset.py` (обработка данных) и `utils.py` (работа модели).
